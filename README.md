## 📌 Motivation

As a developer, understanding design patterns is about more than just "coding"—it's about "designing." This project documents my journey in mastering the **Gang of Four (GoF)** patterns, focusing on how to solve recurring design problems while adhering to **SOLID** principles.

---

## 🚀 Patterns Implemented

The patterns are categorized into three fundamental groups:

### 1. Creational Patterns
*Focus on object creation mechanisms, trying to create objects in a manner suitable to the situation.*
* **Singleton** - Ensuring a class has only one instance.
* **Factory Method** - Providing an interface for creating objects in a superclass.
* **Abstract Factory** - Producing families of related objects.
* **Builder** - Constructing complex objects step-by-step.
* **Prototype** - Creating new objects by copying an existing one.

### 2. Structural Patterns
*Focus on how classes and objects are composed to form larger structures.*
* **Adapter** - Allowing incompatible interfaces to work together.
* **Bridge** - Splitting a large class into two separate hierarchies.
* **Decorator** - Adding new behaviors to objects dynamically.
* **Facade** - Providing a simplified interface to a complex library.

### 3. Behavioral Patterns
*Focus on algorithms and the assignment of responsibilities between objects.*
* **Observer** - A subscription mechanism to notify multiple objects.
* **Strategy** - Defining a family of algorithms and making them interchangeable.
* **Command** - Turning a request into a stand-alone object.
* **State** - Allowing an object to alter its behavior when its internal state changes.

## 📖 How to Explore

Each pattern is organized into its own package. Inside each directory, you will find:
1.  **Interface/Abstract Class:** Defining the contract.
2.  **Concrete Implementations:** The actual logic.
3.  **Client/Demo Class:** A `main` method to demonstrate the pattern in action.
